# サーバー確認ガイド

## 🔍 サーバーが起動しているか確認

### ステップ1: ターミナルを確認

サーバーを起動しているターミナルで、以下のメッセージが表示されているか確認：

```
🚀 Server running on port 3000
📱 WhatsApp webhook: http://localhost:3000/api/whatsapp/webhook
```

**このメッセージが表示されていれば、サーバーは起動しています！**

---

## ✅ 簡単な確認方法

### 方法1: テストエンドポイント（認証不要）

ブラウザで以下を開く：

```
http://localhost:3000/api/test
```

または

```
http://localhost:3000/api/info
```

**これが成功すれば、サーバーは正常に動作しています！**

### 方法2: ヘルスチェック

```
http://localhost:3000/health
```

---

## 📊 統計情報の確認方法

### ステップ1: サーバーが起動していることを確認

上記のテストエンドポイントで確認。

### ステップ2: 統計情報にアクセス

```
http://localhost:3000/api/admin/stats?token=change-this-in-production
```

**重要**: URLの最後に `?token=change-this-in-production` を追加してください。

---

## 🆘 接続できない場合

### 確認事項

1. **サーバーが起動しているか**
   - ターミナルに `🚀 Server running on port XXXX` が表示されているか

2. **正しいポート番号を使っているか**
   - ターミナルの起動メッセージでポート番号を確認
   - ポート3000ではなく、別のポート（3001, 3002など）の可能性

3. **サーバーが停止していないか**
   - ターミナルでエラーが出ていないか
   - `Ctrl+C` を押していないか

### 解決方法

1. **サーバーを再起動**
   ```bash
   # サーバーを停止（Ctrl+C）
   # 再度起動
   npm run dev
   ```

2. **ポート番号を確認**
   - 起動メッセージで実際のポート番号を確認
   - そのポート番号でアクセス

---

## 💡 推奨手順

1. **まずテストエンドポイントで確認**
   ```
   http://localhost:3000/api/test
   ```

2. **成功したら、統計情報を確認**
   ```
   http://localhost:3000/api/admin/stats?token=change-this-in-production
   ```

---

**まずは `/api/test` でサーバーが動いているか確認してください！**
