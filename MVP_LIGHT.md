# 超軽量MVPプラン（2-3週間で動く）

## コンセプト
**外部サービスを最大限活用して、開発工数を最小化**

---

## 構成（超シンプル）

### フロントエンド
- **Tally / Typeform** - ユーザー登録フォーム（開発不要！）
- **簡単なランディングページ** - Next.js（1-2日で完成）

### データ管理
- **Airtable / Google Sheets** - ユーザーデータ管理（開発不要！）
- **Zapier / Make** - 自動化（開発不要！）

### カレンダー
- **Calendly** - カレンダー連携（開発不要！）
- または **Google Calendar** - 手動確認

### 決済
- **Stripe Checkout** - リンク発行のみ（1日で実装可能）

### 通知
- **Email** - SendGrid（1日で実装可能）
- **WhatsApp** - 後回し

### 通話
- **Google Meet** - 手動でリンク共有（最初は人力）

## 具体的な実装方法

### 1. ユーザー登録
```
Tally/Typeform → Airtable/Sheets に自動保存
（Zapier/Makeで自動化）
```

### 2. マッチング
```
Airtable/Sheets のデータを読み込んで
シンプルなマッチングロジック実行
→ 候補を表示
```

### 3. アポイントメント確定
```
Calendly を使う
または
Google Calendar を手動確認して
Google Meet リンクを手動で共有
```

### 4. 決済
```
Stripe Checkout のリンクを発行
→ ポイント付与（Airtable/Sheets に記録）
```

### 5. 通知
```
SendGrid で Email 送信
（Zapier/Makeで自動化可能）
```

## 軽量版のメリット

✅ **開発期間が短い**（3週間 vs 8-10週間）  
✅ **コストが低い**（外部サービス活用）  
✅ **リスクが低い**（既存サービス利用）  
✅ **すぐにテスト可能**（ユーザーに使ってもらえる）  
✅ **柔軟性が高い**（後から改善可能）  

---

## 軽量版のデメリット

❌ **スケーラビリティに限界**（外部サービス依存）  
❌ **カスタマイズ性が低い**（外部サービスの制約）  
❌ **コストが増える可能性**（外部サービス利用料）  
❌ **人力オペレーションが必要**（初期は手動対応）
