# SourceTree（ソースツリー）完全ガイド

## 📚 このガイドについて

SourceTreeは、GitHub Desktopと同様に、Gitを視覚的に操作できるツールです。
このガイドでは、SourceTreeの使い方を非エンジニア向けにわかりやすく説明します。

---

## 🤔 SourceTreeとは？

### 簡単に言うと

**SourceTree = コードをGitHubに保存するためのツール（GUI版）**

- ✅ ボタンをクリックするだけで操作できる
- ✅ コマンドを覚える必要がない
- ✅ 無料
- ✅ MacとWindowsの両方で使える

### GitHub Desktopとの違い

| 機能 | SourceTree | GitHub Desktop |
|------|-----------|----------------|
| 無料 | ✅ | ✅ |
| 使いやすさ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |
| 機能の多さ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| 初心者向け | ⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

**結論**: 初心者の方はGitHub Desktopの方が簡単ですが、SourceTreeも十分使いやすいです！

## 🚀 インストール方法

### ステップ1: SourceTreeをダウンロード（3分）

1. **SourceTreeのサイトを開く**
   - ブラウザで https://www.sourcetreeapp.com を開く

2. **「Download」をクリック**
   - 自動的にMac版またはWindows版が選択されます

3. **ダウンロード**
   - ダウンロードが開始されます
   - 数分かかります

### ステップ2: SourceTreeをインストール（5分）

#### Macの場合

1. **ダウンロードしたファイルを開く**
   - `SourceTree-x.x.x.dmg` というファイルをダブルクリック

2. **アプリをインストール**
   - ウィンドウが開くので、SourceTreeアイコンを「アプリケーション」フォルダにドラッグ

3. **初回起動**
   - 「アプリケーション」からSourceTreeを起動
   - セキュリティ警告が出たら「開く」をクリック

#### Windowsの場合

1. **インストーラーを実行**
   - ダウンロードした `.exe` ファイルをダブルクリック

2. **インストールウィザードに従う**
   - 「次へ」をクリックして進む
   - インストール場所はデフォルトのままでOK

3. **完了**
   - 「完了」をクリック

### ステップ3: 初回セットアップ（5分）

1. **アカウント設定**
   - SourceTreeを起動すると、アカウント設定画面が表示されます
   - **「スキップ」をクリック**（後で設定できます）

2. **Gitの設定**
   - Gitがインストールされているか確認されます
   - インストールされていない場合は、SourceTreeが自動でインストールします

3. **SSHキーの設定（オプション）**
   - 初回は「スキップ」でOK
   - 後で設定できます

**これでインストール完了！**

## 📦 リポジトリを追加する方法

### 方法A: 既存のリポジトリを追加（推奨）

あなたのパソコンにあるコードをSourceTreeで管理する方法です。

1. **「リポジトリを追加」をクリック**
   - SourceTreeを起動
   - 左上の「リポジトリを追加」ボタンをクリック

2. **「ローカルリポジトリを追加」を選択**
   - 左側の「ローカルリポジトリを追加」タブをクリック

3. **フォルダを選択**
   - 「参照」ボタンをクリック
   - `/Users/masakinakata/LanguageMatching` フォルダを選択
   - 「追加」をクリック

4. **完了！**
   - リポジトリが追加されました

### 方法B: GitHubからクローン（新規）

GitHubに既にリポジトリがある場合、それをダウンロードする方法です。

1. **「リポジトリを追加」をクリック**

2. **「URLからクローン」を選択**
   - 左側の「URLからクローン」タブをクリック

3. **GitHubのURLを入力**
   - ソースURL: `https://github.com/YOUR_USERNAME/LanguageMatching.git`
     - `YOUR_USERNAME` をあなたのGitHubユーザー名に置き換える
   - 保存先: `/Users/masakinakata/LanguageMatching` を選択
   - 「クローン」をクリック

4. **認証**
   - GitHubの認証情報を入力
   - 「ログイン」をクリック

5. **完了！**
   - リポジトリがダウンロードされました

## 💾 コードをGitHubに保存する方法

### ステップ1: 変更を確認

1. **SourceTreeを開く**
   - リポジトリが表示されます

2. **「ファイルステータス」を確認**
   - 左側の「ファイルステータス」タブをクリック
   - 変更されたファイルが表示されます

### ステップ2: ステージング（変更を選択）

1. **変更したいファイルを選択**
   - ファイルステータスで、保存したいファイルにチェックを入れる
   - または「すべてをステージング」をクリック

2. **確認**
   - ステージングされたファイルが「ステージング済み」に表示されます

### ステップ3: コミット（保存）

1. **コミットメッセージを入力**
   - 下部のテキストボックスに、変更内容を説明するメッセージを入力
   - 例: 「ユーザー登録機能を追加」
   - 例: 「バグを修正」

2. **「コミット」をクリック**
   - 右下の「コミット」ボタンをクリック
   - コードがローカルに保存されます

### ステップ4: プッシュ（GitHubにアップロード）

1. **「プッシュ」をクリック**
   - 上部の「プッシュ」ボタンをクリック

2. **リモートを選択**
   - 初回の場合、リモート（GitHub）を設定する必要があります
   - 「リモートを追加」をクリック

3. **リモート情報を入力**
   - リモート名: `origin`（デフォルトのままでOK）
   - URL/パス: `https://github.com/YOUR_USERNAME/LanguageMatching.git`
     - `YOUR_USERNAME` をあなたのGitHubユーザー名に置き換える
   - 「OK」をクリック

4. **認証**
   - GitHubの認証情報を入力
   - 「ログイン」をクリック

5. **プッシュ実行**
   - 「プッシュ」ボタンをクリック
   - コードがGitHubにアップロードされます

**これで完了！コードがGitHubに保存されました！**

## 🔄 日常的な作業フロー

### 毎日の作業パターン

1. **コードを編集**
   - パソコンでコードファイルを変更

2. **SourceTreeで確認**
   - SourceTreeを開く
   - 「ファイルステータス」で変更を確認

3. **ステージング**
   - 変更したいファイルを選択

4. **コミット**
   - メッセージを入力して「コミット」

5. **プッシュ**
   - 「プッシュ」をクリックしてGitHubにアップロード

**これで、コードが安全に保存されます！**

## 📥 GitHubから最新のコードを取得する方法

他の人（または別のパソコン）で変更したコードを取得する方法です。

1. **「取得」をクリック**
   - 上部の「取得」ボタンをクリック

2. **リモートを選択**
   - `origin` を選択（通常はこれ）

3. **「取得」をクリック**
   - 最新のコードがダウンロードされます

4. **マージ（統合）**
   - 変更がある場合、「マージ」または「プル」をクリック
   - ローカルのコードと統合されます

## 🔍 SourceTreeの主要機能

### 履歴の確認

1. **「履歴」タブをクリック**
   - 左側の「履歴」タブをクリック
   - 過去のコミット履歴が表示されます

2. **過去のバージョンを確認**
   - コミットをクリックすると、その時点のコードを確認できます

### ブランチの作成・切り替え

1. **「ブランチ」をクリック**
   - 上部の「ブランチ」ボタンをクリック

2. **新しいブランチを作成**
   - ブランチ名を入力
   - 「ブランチを作成」をクリック

3. **ブランチを切り替え**
   - ブランチ一覧から選択

**ブランチ = コードの別バージョンを作る機能**

## ⚙️ GitHubアカウントの連携

### 初回設定

1. **「設定」を開く**
   - SourceTree → 「設定」（Mac）または「ツール」→「オプション」（Windows）

2. **「アカウント」タブをクリック**

3. **「追加」をクリック**

4. **GitHubを選択**
   - 「ホスティングサービス」で「GitHub」を選択

5. **認証情報を入力**
   - ユーザー名とパスワード（またはPersonal Access Token）を入力
   - 「OK」をクリック

**これで、GitHubとの連携が完了しました！**

## 🆘 よくある問題と解決方法

### Q: 「認証に失敗しました」エラー

**A: Personal Access Tokenを使用する**

1. GitHubでPersonal Access Tokenを作成
   - https://github.com/settings/tokens にアクセス
   - 「Generate new token」をクリック
   - スコープで `repo` にチェック
   - トークンをコピー

2. SourceTreeでトークンを使用
   - パスワードの代わりに、Personal Access Tokenを入力

### Q: 「リモートが見つかりません」エラー

**A: リモートを追加する**

1. 「設定」→「リモート」を開く
2. 「追加」をクリック
3. リモート名: `origin`
4. URL: `https://github.com/YOUR_USERNAME/LanguageMatching.git`
5. 「OK」をクリック

### Q: 「プッシュが拒否されました」エラー

**A: まずプル（取得）する**

1. 「取得」をクリック
2. 「プル」をクリック
3. 再度「プッシュ」を試す

### Q: ファイルが表示されない

**A: リフレッシュする**

1. 「ファイルステータス」タブをクリック
2. または、SourceTreeを再起動
