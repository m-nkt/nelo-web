# 超わかりやすい！中学生でもわかる手順

## 🎯 目標

サーバーが動いているか確認して、統計情報を見ること

---

## 📝 手順（1つずつ順番に）

### ステップ1: サーバーが動いているか確認

#### 1-1. サーバーを起動しているターミナルを確認

1. **ターミナルを開く**
   - Macの「ターミナル」アプリを開く

2. **サーバーを起動しているターミナルを見つける**
   - サーバーを起動したターミナルウィンドウを探す
   - 以下のような文字が表示されているはず：
     ```
     🚀 Server running on port 3000
     ```

3. **このメッセージが表示されていればOK！**
   - 表示されていなければ、サーバーを起動する必要があります

#### 1-2. サーバーを起動する（まだの場合）

1. **ターミナルで以下を入力：**
   ```bash
   cd /Users/masakinakata/LanguageMatching
   ```

2. **Enterキーを押す**

3. **次に以下を入力：**
   ```bash
   npm run dev
   ```

4. **Enterキーを押す**

5. **以下のメッセージが表示されるまで待つ：**
   ```
   🚀 Server running on port 3000
   ```
   
   **これが表示されれば、サーバーは起動しています！**

---

### ステップ2: ブラウザで確認

#### 2-1. ブラウザを開く

1. **Safari、Chrome、Firefoxなど、好きなブラウザを開く**

2. **アドレスバーをクリック**
   - ブラウザの上の方にある、URLを入力する場所をクリック

#### 2-2. テストページを開く

1. **以下の文字をコピー：**
   ```
   http://localhost:3000/api/test
   ```

2. **ブラウザのアドレスバーに貼り付け**
   - コピーした文字を、アドレスバーに貼り付ける（Command+V）

3. **Enterキーを押す**

4. **結果を確認**
   - 以下のような文字が表示されれば成功：
     ```json
     {
       "message": "Server is working!",
       "timestamp": "...",
       "port": 3000
     }
     ```
   
   **これが表示されれば、サーバーは正常に動作しています！** ✅

---

### ステップ3: 統計情報を見る

#### 3-1. 統計情報のページを開く

1. **ブラウザのアドレスバーをクリック**

2. **以下の文字をコピー：**
   ```
   http://localhost:3000/api/admin/stats?token=change-this-in-production
   ```

3. **アドレスバーに貼り付け**

4. **Enterキーを押す**

5. **結果を確認**
   - 以下のような文字が表示されれば成功：
     ```json
     {
       "error": "Database is not configured",
       "stats": {
         "totalUsers": 0,
         "totalAppointments": 0,
         ...
       }
     }
     ```
   
   **これが表示されれば成功です！** ✅
   
   （データベースが設定されていないので、数字は0ですが、これは正常です）

## 🆘 うまくいかない場合

### 問題1: 「接続できません」と表示される

**原因**: サーバーが起動していない可能性

**解決方法**:
1. ターミナルで `🚀 Server running on port 3000` が表示されているか確認
2. 表示されていなければ、ステップ1-2でサーバーを起動

### 問題2: ポート番号が違う

**原因**: サーバーが別のポート（3001, 3002など）で起動している

**解決方法**:
1. ターミナルの起動メッセージを確認
2. `🚀 Server running on port 3001` と表示されていたら、URLの `3000` を `3001` に変更
   - 例: `http://localhost:3001/api/test`

### 問題3: 「Unauthorized」と表示される

**原因**: 認証トークンが間違っている

**解決方法**:
- URLの最後に `?token=change-this-in-production` が含まれているか確認
- 含まれていない場合は、URLの最後に追加

## 🎓 用語説明（わからない言葉があったら）

- **サーバー**: プログラムが動いているコンピューター（今はあなたのMac）
- **ポート**: サーバーに接続するための番号（例: 3000）
- **URL**: ブラウザのアドレスバーに入力する文字（例: http://localhost:3000）
- **JSON**: データを表示する形式（{ "key": "value" } のような形）
