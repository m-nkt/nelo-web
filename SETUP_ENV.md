# .envファイルの設定ガイド

## 🎯 このガイドの目的

`.env`ファイルを設定して、エラーを解決する方法を説明します。

---

## ⚠️ 現在のエラー

```
Error: supabaseUrl is required.
```

このエラーは、`.env`ファイルにSupabaseの設定がないことが原因です。

---

## ✅ 解決方法（2つの選択肢）

### 選択肢1: とりあえずサーバーを起動する（簡単）

**今すぐサーバーを起動したい場合**:

1. `.env`ファイルは既に作成されています（空欄のまま）
2. サーバーは起動しますが、データベース機能は使えません
3. 後でSupabaseを設定すれば、すべての機能が使えます

**この方法で進める場合**: 何もする必要はありません。そのまま `npm run dev` を実行してください。

---

### 選択肢2: Supabaseを設定する（推奨）

**すべての機能を使いたい場合**:

#### ステップ1: Supabaseアカウントを作成（5分）

1. **Supabaseのサイトを開く**
   - https://supabase.com にアクセス

2. **アカウントを作成**
   - 「Start your project」をクリック
   - GitHubアカウントでログイン（推奨）

3. **プロジェクトを作成**
   - 「New Project」をクリック
   - プロジェクト名を入力（例: `language-matching`）
   - データベースのパスワードを設定（**メモしておく！**）
   - リージョンを選択（`Tokyo` を推奨）
   - 「Create new project」をクリック

4. **プロジェクトの作成を待つ**
   - 数分かかります

#### ステップ2: 認証情報を取得（3分）

1. **左メニューから「Settings」→「API」をクリック**

2. **認証情報をコピー**
   - `Project URL` をコピー
   - `anon public` キーをコピー
   - `service_role` キーをコピー（**重要: これは秘密にしてください**）

#### ステップ3: .envファイルを編集（2分）

1. **`.env`ファイルを開く**
   - `/Users/masakinakata/LanguageMatching/.env` をテキストエディタで開く

2. **以下の値を設定**
   ```
   SUPABASE_URL=ここにProject URLを貼り付け
   SUPABASE_KEY=ここにanon publicキーを貼り付け
   SUPABASE_SERVICE_KEY=ここにservice_roleキーを貼り付け
   ```

3. **保存**

#### ステップ4: データベーススキーマを実行（5分）

1. **Supabaseで「SQL Editor」を開く**
   - 左メニューから「SQL Editor」をクリック

2. **「New query」をクリック**

3. **スキーマを実行**
   - `/Users/masakinakata/LanguageMatching/database/schema.sql` を開く
   - 内容をすべてコピー
   - SQL Editorにペースト
   - 「Run」をクリック

4. **成功メッセージを確認**
   - 「Success. No rows returned」と表示されればOK

---

## 🚀 サーバーを起動

設定が完了したら、サーバーを起動してください：

```bash
npm run dev
```

成功メッセージが表示されればOKです！
